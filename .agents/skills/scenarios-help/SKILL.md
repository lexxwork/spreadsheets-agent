---
name: scenarios-help
description: Use when the user asks how to run scenarios, what ready-made scenarios exist, how CLI arguments work, or how to use scenarios through an agent
---

When invoked:
1. Read `docs/scenarios.md`.
2. Read `docs/usage.md`.
3. List local `scenarios/*.js` files.

Summarize:
- available scenarios and what each one does
- how to run them from CLI
- how to use the repository through MCP and an agent
- which scenario is safest to try first
- that `write-test` expects a local `configs/write-test.json` with a real `spreadsheetId` and `sheetName`
